# M6 Spec — Loop Web UI + 自然语言创建 + 触发 + Review

> **Status:** 🏗 Design → Implementation
> **Baseline:** M5（LOOP.md + `/api/loops` CRUD）完成态。

**Goal:** Web UI + 自然语言创建 Loop + 手动触发 + Review API。MVP 收尾。

---

## 1. 自然语言创建

```
POST /api/loops
  输入: { intent, projectId }
  流程:
    1. backend: 创建 cron_job 行（cronExpr=""）
    2. backend: mkdir loops/<name>/skills/
    3. backend: 起 AgentSession（loop-config-generator skill）
       → 系统 prompt 含 registry.yaml
       → agent 输入: intent + targetDir
       → agent 用 write 工具创建 LOOP.md + 复制 skills
       → agent 用 update_loop_config 工具设 cronExpr
    4. agent 结束 → backend 读 LOOP.md → 返回预览
  输出: { loop: { id, name, cronExpr, loopConfigPath, preview } }
```

### 新工具: `update_loop_config`

```typescript
{
  name: "update_loop_config",
  description: "Set the cron schedule for this Loop",
  inputSchema: {
    cronExpr: string,  // 5-field, backend 校验
  },
  execute: async (input) => {
    assertValidCronExpr(input.cronExpr);
    port.updateCronJob(cronJobId, { cronExpr: input.cronExpr });
    scheduler.reregister(cronJobId);  // stop old + register new
  }
}
```

## 2. 手动触发

```
POST /api/loops/:id/run
  → loopStep({ loopConfigPath, sessionFactory, buildSpec })
  → 返回新 state
```

## 3. Review API

```
POST /api/loops/:id/review
  输入: { itemId, verdict: "approve"|"reject"|"promote"|"retry"|"dismiss", feedback? }
  → loopStep({ loopConfigPath, sessionFactory, buildSpec, action })
  → 返回新 state
```

## 4. 写锁

loopStep() 入口加 per-loop 排他锁（文件锁 `Bun.file(lockPath).lock()`）：

```typescript
const lockPath = `${params.loopConfigPath}/LOCK`;
const lock = Bun.file(lockPath);

// 拿锁（阻塞直到可用）
await lock.lock();

try {
  // ... loopStep body
} finally {
  await lock.unlock();
}
```

三条入口（cron/manual/review）共用同一把文件锁。

## 5. Web UI

### `/loops` — 仪表盘

卡片列表。每卡：name、一行描述、schedule、pending count badge、[Pause/Resume] [Run Now] [View]。

参照 `apps/web/src/app/(main)/cron/page.tsx` 的风格。

### `/loops/new` — 创建

一个输入框（intent）+ Project 下拉 → Submit 调 `POST /api/loops` → 显示预览 → 确认后跳转 `/loops/:id`。

### `/loops/:id` — 详情

显示 lastRun、pendingCount。Review queue 列出 `step=awaiting_review` 的 item，每条有 [Approve] / [Reject] 按钮（调 POST `/api/loops/:id/review`）。

## 6. 导航

NavRail.tsx：加 "Loops"（`<RefreshCwIcon />`），删 "Issues"。

## 7. 验收标准

### 后端

1. **`update_loop_config` 工具**：agent 可调用，cronExpr 被校验，scheduler 重注册
2. **POST `/api/loops` 自然语言创建**：起 AgentSession 写 LOOP.md + 设 cronExpr
3. **POST `/api/loops/:id/run`**：手动触发 loopStep，返回新 state
4. **POST `/api/loops/:id/review`**：带 action 调 loopStep，返回新 state
5. **写锁**：loopStep 入口拿文件锁，同一 Loop 串行

### 前端

6. **`/loops` 仪表盘**：卡片列表，pending count badge
7. **`/loops/new` 创建页**：intent 输入 + Project 下拉 → 预览 → 确认
8. **`/loops/:id` 详情页**：pending items + [Approve]/[Reject] 按钮
9. **Pause/Resume**：卡片 Switch 调 PATCH /api/cron-jobs/:id
10. **导航**：Loops 入口，Issues 移除
11. **全 workspace typecheck + lint + test 通过**
