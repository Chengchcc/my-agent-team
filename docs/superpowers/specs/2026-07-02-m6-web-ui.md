# M6 Spec — Loop Web UI

> **Status:** 🏗 Design → Implementation
> **Baseline:** M5（LOOP.md + `/api/loops` CRUD）完成态。
> **关联:** `apps/web/src/app/(main)/cron/page.tsx`（参考页面）· `apps/web/src/components/NavRail.tsx`（导航）· `apps/web/src/lib/api.ts`（API client）

**Goal:** 三个前端页面 + 导航变更。MVP 收尾——用户能从 Web 管理 Loop。

**Non-goals:**
- 不做自然语言创建（M5.1，loop-config-generator Skill 接线）
- 不做 review 交互（M5.1）
- 不做运行历史时间线（M5.1）
- 不做预算可视化（M5.1）

```
└──────────────────────────────────────────────────────────────┘
```

### 交互
- **[+ Create Loop]** → 跳转 `/loops/new`
- **[Pause/Resume]** → toggle enabled → `PATCH /api/cron-jobs/:id { enabled }`
- **[Run Now]** → placeholder（M5.1 手动触发）
- **[View →]** → 跳转 `/loops/:id`
- 空状态："No loops yet. Create your first loop to automate work."

## 4. `/loops/new` — 创建表单

```
┌───────────────────────────────────────────────────────────┐
│ Create Loop                                               │
│                                                           │
│ Name *          [Morning Triage                    ]      │
│ Project *       [my-app ▼]                                │
│ Schedule        [Daily 8:00 AM            ▼]              │
│ Intent          [每天早上检查 CI 失败，自动修简单的  ]    │
│                                                           │
│                        [Cancel]  [Create Loop]            │
└───────────────────────────────────────────────────────────┘
```

### 字段
- **Name**（必填）：Loop 名称
- **Project**（必填）：从 `GET /api/projects` 加载下拉
- **Schedule**：预设选项（Daily 8am / Every 15m / Every 2h / Every 6h / Manual）或自定义 cron
- **Intent**：自然语言描述（M5.1 做翻译，现在只存文本）

### Submit
`POST /api/loops { name, projectId, cronExpr, intent }` → 成功后跳转 `/loops`

## 7. 验收标准

1. **`/loops` 页面可访问**：GET /api/loops 返回的数据正确渲染
2. **卡片显示 pending count**：`GET /api/loops/:id` 的 pendingCount 显示为 Badge
3. **空状态**：无 Loop 时显示引导文字
4. **`/loops/new` 创建表单**：Project 下拉从 API 加载，Submit 调 POST /api/loops
5. **`/loops/:id` 详情页**：显示 lastRun、pendingCount、review queue（STATE.md 的 awaiting_review items）
6. **Pause/Resume toggle**：卡片上的 Switch 调 PATCH /api/cron-jobs/:id
7. **导航：Loops 入口**：NavRail 有 Loops 链接，高亮当前路由
8. **导航：Issues 移除**：NavRail 不再显示 Issues
9. **全 workspace typecheck + lint + test 通过**
