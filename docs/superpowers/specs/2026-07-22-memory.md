# Spec: Memory Plugin 重构 + Autonomous Memory

## Problem

当前 memory 功能有三个问题：

1. **手动提取** — agent 必须主动调用 `memory_write` 才能记东西，不调用就失忆
2. **路径不统一** — `fsMemoryPlugin` 读共享 workspace，`agent-identity` 读 per-agent 目录
3. **搜索弱** — `memory_search` 纯文本 grep，多词搜索失效，无时间过滤
4. **所有权分散** — memory 职责分散在 `fs-memory` / `agent-identity` / `conversation-compose` 三处

## Goal

统一 memory 所有权到 `plugin-memory` 包，实现自动提取 + 批量存储 + 智能搜索。

## Design

### 包结构

```
packages/plugin-memory/          ← plugin-fs-memory 改名
├── memory-plugin.ts             ← fs-memory.ts 改名，加 autoMemory + cwd 切换
├── auto-memory.ts               ← 新：extractMemories() + consolidateMemories()
├── memory-retain.ts             ← memory-write.ts 升级（批量 + context）
├── memory-search.ts             ← 升级（多词 AND + 时间过滤）
├── memory-read.ts               ← 不变
└── prompts.ts                   ← 新：OMP stage_one + consolidation prompts
```

### 工具升级

#### memory_retain（替代 memory_write）

```typescript
// Schema
{
  items: [{
    content: string,       // 记忆内容
    context?: string,      // 来源上下文（文件名/函数名/场景）
    tags?: string[],       // 标签
  }]
}
// 存储: memory/facts/<ISO-TS>-<slug>.md（同现有格式）
```

#### memory_search 升级

```typescript
// Schema
{
  query: string,           // 自然语言
  limit?: number,          // 默认 5
  since?: string,          // "7d" / "24h" / ISO date
}

// 算法
1. 分词 query → tokens
2. 时间过滤（since → ISO 前缀比较）
3. 遍历 facts/ 文件：
   - 读 frontmatter + body
   - 所有 tokens 都命中 content 或 context → 记分（命中密度）
   - 部分 tokens 命中 → 降权
4. 按得分排序，返回 top N + 匹配片段
```

### Autonomous Memory pipeline

```
afterModel（每 run 结束）
  ↓
convPort.getLedgerEntries(cid, { sinceSeq: state.lastExtractedSeq })
  ↓ 新条目 >= minMessagesForExtraction?
  ↓ yes
Stage 1: 小模型提取
  input: conversation messages + stage_one prompt
  output: JSON [{ content, context, tags }]
  ↓ 非空
  写入 memory/facts/<ts>-<slug>.md（每一条一个文件）
  ↓ 累积 facts >= consolidateThreshold?
  ↓ yes
Phase 2: LLM 合并
  input: 所有 facts + consolidation prompt
  output: { memory_md, memory_summary }
  ↓
  写入 MEMORY.md + memory_summary.md
  ↓
  lastExtractedSeq = 最新 seq
```

### Settings 配置

| Key | 默认值 | 说明 |
|-----|--------|------|
| `memory.autoExtract` | `true` | 启用自动提取 |
| `memory.extractProvider` | `"anthropic"` | Stage 1 模型 provider |
| `memory.extractModel` | `"claude-haiku-3-5"` | Stage 1 模型（小/便宜） |
| `memory.consolidateProvider` | `"anthropic"` | Phase 2 模型 provider |
| `memory.consolidateModel` | `"claude-sonnet-4-6"` | Phase 2 模型（大/聪明） |
| `memory.minMessagesForExtraction` | `5` | 至少 N 条新消息才触发 Stage 1 |
| `memory.consolidateThreshold` | `10` | 累积 N 条 facts 触发 Phase 2 |

### 注入

```
beforeRun:
  read memory_summary.md（per-agent cwd）→ MemoryKey context
  → metaContext 注入 <memory> 标签（已有）
```
