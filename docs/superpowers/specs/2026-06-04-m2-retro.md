## Actual Implementation vs Spec

### Matched

- `AnthropicChatModel` wraps `@anthropic-ai/sdk`, translates SSE events into `AIMessageChunk`
- System message extracted from messages array, passed as API `system` parameter
- `toAnthropicTools()` pure field mapping: name/description → name/description, inputSchema → input_schema
- Error passthrough (no wrapping)
- Constructor config: model, apiKey, baseUrl, maxTokens, thinking
- 6 tools: web_fetch, web_search (Tavily factory), memory_save/memory_recall (Map factory), read, write
- CLI reads env vars, parses --model/--max-steps/--system, loops readline → run() → stream output
- All tests colocated, mock SDK for adapter, mock fetch for HTTP tools, real files for read/write

### Differences

- `AnthropicChatModel` uses SDK's `MessageParam` and `ContentBlockParam` types for message construction instead of `as any`
- `InputJSONDelta` in the real SDK doesn't carry `id`; adapter maintains an index → id map from `content_block_start` events
- `finalMessage()` is async (returns `Promise<Message>`), not a property — adapter awaits it
- Adapter build tsconfig includes `bun-types` (needs `process.env`)
- Tools-common build tsconfig includes `bun-types` (needs `Bun.file()`/`Bun.write()`)
- `tsconfig.eslint.json` includes `apps/**/*.ts` for ESLint project service
- CLI doesn't use `packages/.gitkeep` pattern; `apps/` dir already existed
- SDK upgraded from 0.54.0 to 0.100.1 to support `thinking: { type: "adaptive" }` and `effort` params
- System message ContentBlock[] extraction implemented (first text block)

## Lessons

- Anthropic SDK `InputJSONDelta` doesn't carry `tool_use_id` — need index-based mapping from `content_block_start` events
- `finalMessage()` is async, not a sync property — pipe it through `await`
- Packages that use Bun-native APIs (Bun.file, Bun.write, process.env) need `bun-types` in their build tsconfig, not just test tsconfig
- Mocking `globalThis.fetch` in Bun requires `as unknown as typeof fetch` because `Mock<>` doesn't satisfy the full `typeof fetch` interface
- `mock.module` from Bun works well for SDK mocking — no need for a separate mocking library
