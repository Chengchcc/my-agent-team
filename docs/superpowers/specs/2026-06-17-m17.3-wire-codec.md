# M17.3 Wire Codec — 让每种线上形状自描述、单一所有者、可独立审计

> **Status:** Spec → Ready for implementation.
> **Baseline commit:** `origin/master` @ `59236aa8`.
> **关联:** `docs/architecture/architecture-design-philosophy.md` · `packages/message/src/parser.ts` · `packages/runner-protocol/src/{messages,ndjson}.ts` · `packages/event-log/src/index.ts` · `apps/web/src/hooks/useConversation.ts` · `apps/lark-bot/src/sse-watcher.ts`.

## Summary

M17.1/M17.2 把消息语义下压成唯一 `Message` 本体、把 `AgentEvent` 收敛成 `MessageRevision` 流、把 `LedgerEntry.content` 收紧成字符串。M17.3 处理机制层脏点：每种 wire shape 在拼装处和解析处各写一遍，中间没有运行时校验、收发共用的 codec。

不是新增领域概念——是把已隐式存在的传输契约显式化、收口到单一所有者。每种 wire shape 一个 zod codec，schema 为唯一 source of truth。

## 目标

1. C1 `MessageRevision` — 重写手写 parser 为 zod（签名不变）
2. C2 `AgentEvent` — 建 zod schema，删除 `error` 死变体
3. C3 runner 帧 — `RunnerToHost`/`HostToRunner` zod schema
4. C4 `LedgerEntry`/`LedgerKind` — 收敛为单一 zod 定义

## 实施顺序

1. C1: message codec 改 zod
2. C4: LedgerEntry/LedgerKind 收敛
3. C2: AgentEvent codec + 删 error
4. C3: runner 帧 codec + ndjson 边界校验
5. web/lark 消费端收口
6. 写边界统一走 codec.serialize
7. 清 as 断言 + 文档
