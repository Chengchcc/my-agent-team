# M17.2 E2E Message Envelope — 一条消息只组装一次

> **Status:** Spec → Ready for implementation（已按基线 `a4099b6` 实测校正）。
> **Baseline commit:** `origin/master` @ `a4099b6`
> **关联:** `docs/architecture/design-philosophy.md` · `packages/framework/src/create-agent.ts` · `packages/runner-protocol/src/messages.ts` · `apps/backend/src/main.ts` · `apps/backend/src/features/run/supervisor.ts` · `apps/web/src/lib/conversation-reducer.ts` · `apps/lark-bot/src/sse-watcher.ts`

## Summary

M17.1 统一了类型。M17.2 统一流动：让一条消息在它诞生的地方（framework）成为一个 `MessageRevision` 流，沿途每一跳只**透传 + 盖戳**，不再重组、不再用侧信道缝合、不再维护多套表示。

四层漂移收敛：
- **L1 信封漂移**：删 `buildAssistantRevision()`，组装唯一发生在 framework
- **L2 词表漂移**：`AgentEvent` 收敛为消息/observability/control 三类，`text_delta`/`tool_start`/`tool_end` 吸收进 revision 流
- **L3 observability**：删除 `tool_start`/`tool_end` 第三套表示，工具渲染读 `tools[]`、指标读 `tool_call`
- **L4 投影**：`LedgerEntry.content` 从 `z.unknown()` 收敛为序列化 `MessageRevision`，删形状猜测，去渲染前缀

## 实施顺序

1. Commit 1: framework 收敛词表 + emit MessageRevision (L1/L2 核心)
2. Commit 2: runner 透传 + 通道收敛
3. Commit 3: backend 去组装
4. Commit 4: Web/Lark 回归（不改结构）
5. Commit 6: 账本去 unknown + 投影同源 (L4)
6. Commit 7: observability 收敛 (L3)
7. Commit 8: 清理 + 边界测试
8. Commit 9: 文档跟随
