# M8 Spec — Backend MVP（Team Runtime HTTP Server + SQLite Checkpointer）

> M8 把架构文档里定型的 **L5 Backend**（[11-backend](../architecture/11-backend.md)）从纸面变成可跑的 `apps/backend`：一个 Bun HTTP server，管理 agent / thread / run CRUD，用 SQLite 持久化**元数据 + checkpointer messages**，复用 M7 runner-stdio 内核 in-proc 跑 agent，通过 SSE 把 `AgentEvent` 流式推给客户端。

> 同时新增 `@my-agent-team/checkpointer-sqlite`，把 harness 默认 checkpointer 从 `inMemory` 切到 sqlite，清理 3 个空壳 sqlite plugin 包，修 M7 N1 bug。

## 一、定位

M8 是 **一个新 app + 一个新 checkpointer 包 + harness 默认行为切换 + 清理积压**：

- 新增 `apps/backend` — Bun.serve HTTP 进程，SQLite 持久化（共库），in-proc runner，SSE 流
- 新增 `@my-agent-team/checkpointer-sqlite` — `Checkpointer` 接口的 sqlite 实现，schema 由该包定义，可被 backend 装载到同一 db
- 改 `@my-agent-team/harness` — 默认 checkpointer 从 `inMemoryCheckpointer()` 切到 sqlite（接受 `dbPath` 入参）
- 改 `@my-agent-team/runner-stdio` — 修 M7 N1 bug（factory 调用从 try 外移到 try 内），同时 runner 暴露"接受外部注入 checkpointer"的口子，让 backend 能传一个 backend-owned checkpointer 进去
- 复用 `@my-agent-team/agent-spec` 作为内部装配契约（即便不跨进程，也走同一 schema 防御性 validate）
- 删 3 个空壳包：`packages/plugin-history-sqlite/` / `packages/plugin-event-log-sqlite/` / `packages/plugin-permission-sqlite/`
- 新增架构文档 [04-checkpointer §已知限制](../architecture/04-checkpointer.md) 说明 sandbox 隔离下文件型 checkpointer 失效，长期方向是 backend 内部子服务

**M8 显式不做**：

- **不 spawn runner 子进程** — in-proc 模式跑足够；stdio spawn 是 M9
- **不实现 plugin-permission** — `permissionMode` 字段先持久化、传 harness，但 harness 侧无 enforcement 是 M8.5
- **不接 frontend** — 仅暴露 HTTP/SSE，frontend SPA 是 M13
- **不接 IM bot** — 仅暴露 HTTP/SSE，IM adapter 是 M14
- **不引入 Member / Conversation / Task 抽象** — schema 字段为它们留口子（`threads.kind`），代码只支持 `'conversation'`
- **不做沙箱** — workspace 直接 mkdir 本机文件系统；checkpointer 走 sqlite 文件路径（这层在 sandbox 启用前必须升级为 HTTP/RPC，见 04-checkpointer 已知限制）
- **不做 Checkpointer HTTP server 化** — sandbox 没启用前，in-proc / 同机子进程都能直接 open sqlite 文件，HTTP 化是 future work
- **不做多实例 / load balancer** — 单进程 server
- **不做鉴权进化** — 单 `X-Auth-Token` 头匹配 config 里的固定 token
- **不做显式 abort 端点** — 客户端断 SSE 连接即 abort；显式 `POST /runs/:id/cancel` 加入，**作为兜底**（防 EventSource 重连误触发）

## 三、设计原则

沿用 M1–M7 不变，强化三条：

- **第一性原理** — backend 是"用 HTTP 把 harness 装出去"的最小载体；任何能在 harness/framework 解决的事不下沉到 backend
- **奥卡姆剃刀** — in-proc runner（直接调 `runEntry`）足够，不抽 runner 抽象层；SQLite native `bun:sqlite`，不引入 ORM；Bun.serve native 路由，不引入 hono/express
- **Vertical Slice + Hexagonal**（grill A2）— feature 内聚（`features/agent` / `features/thread` / `features/run` / `features/checkpoint`），每个 feature 内部走 ports & adapters；composition root 在 `main.ts`
- **State belongs to backend** — agent / thread / run 元数据 + messages 历史全部由 backend 进程持有（共库 `backend.db`）
- **契约独立于双方** — backend ↔ runner 继续走 `AgentSpec` schema；checkpointer 通过 `CheckpointerPort` 抽象，未来 sandbox 时切 HTTP adapter

## 四、交付范围

### 4.1 新包：`@my-agent-team/checkpointer-sqlite`

**包路径**：`packages/checkpointer-sqlite/`

**依赖**：
- `@my-agent-team/framework`（type-only：`Checkpointer` / `Message` / `InterruptState` / `CheckpointEvent`）
- **零运行时依赖**（`bun:sqlite` 是 Bun built-in，不算 npm dep）

**导出**：

```ts
// packages/checkpointer-sqlite/src/index.ts
import type { Database } from "bun:sqlite";
import type { Checkpointer } from "@my-agent-team/framework";

export interface SqliteCheckpointerOptions {
  db: Database | string;
  tablePrefix?: string;
}

export function sqliteCheckpointer(opts: SqliteCheckpointerOptions): Checkpointer;
export const SQLITE_CHECKPOINTER_MIGRATIONS: readonly { id: number; up: string }[];
```

### 4.2 改：`@my-agent-team/harness`

默认 checkpointer 从 `inMemoryCheckpointer()` 切到 sqlite；接受 `checkpointer: Checkpointer | "memory" | "sqlite"` + `checkpointerDb?: Database`。

### 4.3 改：`@my-agent-team/runner-stdio`

修 M7 N1 bug：factory 调用进 try。`EntryIO` 加 `checkpointerDb?: Database` 字段。

### 4.4 删：3 个空壳 sqlite plugin 包

plugin-history-sqlite / plugin-event-log-sqlite / plugin-permission-sqlite。当前这些目录不存在于磁盘——commit 1 跳过删除。

### 4.5 新 app：`apps/backend`

Bun.serve HTTP 进程。Vertical Slice + Hexagonal 目录结构。SQLite 持久化（共库 `backend.db`）。In-proc runner（通过 `EntryIO.checkpointerDb` 注入 db）。

### 4.6 HTTP API

## 五、技术契约不做（invariant）

- Backend 不持久化 messages 写侧 — 写由 harness 内置 sqliteCheckpointer 直接 INSERT
- Backend 不直接调 framework / core — 通过 harness + runner-stdio 间接消费
- Backend 不引入 ORM — 直接 `bun:sqlite` SQL
- Backend 不引入 HTTP 框架 — Bun.serve native
- 每次 run 走全 spec 装配 + zod parse
- Run 不在 thread 之外发起
- runner-stdio 不直接 import `bun:sqlite`
- checkpointer-sqlite 不依赖 backend
