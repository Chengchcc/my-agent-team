# Spec: Recap Panel（每轮对话实时摘要）

## Problem

用户在长对话中容易丢失上下文——"之前做了什么？现在到哪了？"需要往回滚动查看。pi-recap 提供了常驻的每轮摘要面板解决这个问题。

## Goal

在 ConversationCanvas 右侧添加常驻 RecapPanel，每轮结束后用 cheap model 生成一句话摘要，通过 SSE 实时推送到前端。不持久化，全内存/瞬态。

## Design

### 数据流

```
afterModel（每轮结束）
  → recapModel.stream([conversation_history + recap_prompt])
  → 一句话文本
  → ctx.emit({ type: "recap_update", payload: { text, turn } })
  → conversation SSE → RecapPanel
```

### Plugin 设计

```typescript
// packages/plugin-recap/src/recap-plugin.ts

export interface RecapPluginOptions {
  recapModel: ChatModel;
  enabled?: boolean;
}

export function recapPlugin(opts: RecapPluginOptions): Plugin {
  let turnCount = 0;

  return definePlugin({
    name: "recap",
    hooks: {
      async beforeRun(_ctx, messages) {
        turnCount = 0;
        return [...messages];
      },
      async afterModel(ctx, messages) {
        if (!opts.enabled) return;
        turnCount++;
        const text = await generateRecap(opts.recapModel, messages);
        if (text) {
          ctx.emit?.({
            type: "recap_update",
            payload: { text, turn: turnCount },
          });
        }
      },
    },
  });
}
```

### Settings 配置

| Key | 默认值 | 说明 |
|-----|--------|------|
| `recap.enabled` | `true` | 开关 |
| `recap.provider` | `"anthropic"` | provider id |
| `recap.model` | `"claude-haiku-3-5"` | 最小/最便宜模型 |

RecapPanel：右 260px，常驻，显示最近一轮 recap 文本，暗色半透明背景。turn 数变化时 200ms 淡入动画。

### 不做

- 跨 session 持久化（纯瞬态）
- Goal 自动推导（recap 文本本身包含目标信息）
- Turn history 翻页（只显示最新一轮）
- 键盘导航 / slash command
