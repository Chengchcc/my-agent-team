# M1 Retro — Core Protocol and Minimal Runtime

## Delivered

- Added `@my-agent-team/core` with L1 protocol types and the L2 `run()` loop.
- Added `@my-agent-team/test-helpers` with `echoModel()` for scripted model tests.
- Added colocated Bun tests under `src/*.test.ts`.
- Added `docs/architecture/00-overview.md` with the four-layer model and design discipline.
- Removed `packages/.gitkeep` after real packages were created.

## Actual Implementation vs Spec

### Matched

- `Message`, `ContentBlock`, `ChatModel`, `AIMessageChunk`, and `Tool` are plain TypeScript contracts.
- Tool result blocks use Anthropic-aligned `tool_use_id` and `is_error`.
- `ChatModel` exposes only `stream()`.
- `run()` keeps state in the caller-owned `messages` array.
- `run()` yields `AsyncIterable<Message>` snapshots and completed tool result messages.
- Tool execution is serial in M1.
- Missing tools and thrown tool errors become error `tool_result` blocks.
- Model stream errors are not wrapped.
- Abort avoids pushing partial assistant/tool-result state into `messages`.
- Empty streams do not push empty assistant messages.
- Tests are colocated with source files.

### Differences

- `core` tests use a local `scriptedModel()` instead of importing `@my-agent-team/test-helpers`. This avoids a reverse dependency from `core` to its helper package.
- `turbo.json` typecheck now depends on `^build` as well as `^typecheck`, because workspace package export types resolve through built `dist` files.
- `tsconfig.base.json` includes `DOM` so Web-standard `AbortSignal`, `AbortController`, and `structuredClone` are available in package code and tests.
- TypeScript source imports use explicit `.js` extensions because the repo uses `moduleResolution: "NodeNext"`.
- `run()` yields only when a chunk changes visible assistant content. Done-only chunks do not produce duplicate snapshots.

## Tooling Changes

- `_template/package/tsconfig.test.json` includes `types: ["bun-types"]` for colocated Bun tests.
- Package `typecheck` scripts run `tsc -p tsconfig.test.json --noEmit` so tests are typechecked with source.
- Build `tsconfig.json` excludes `src/**/*.test.ts` so tests are not emitted.

## Lessons

- Keep `core` independent from `test-helpers`; helper packages can depend on core, not the reverse.
- With NodeNext, write `.js` in TypeScript relative imports from the start.
- Workspace packages that export only `dist` need dependency builds before dependent typechecks.
- Medium-grained behavior tests were enough to cover the runtime contract without creating brittle coverage-oriented microtests.
- The retro is part of the delivery, not an afterthought, because it records spec-to-implementation drift for later milestones.
