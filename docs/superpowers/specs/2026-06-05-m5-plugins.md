## 三、设计原则

沿用 M1–M4 不变。

- **第一性原理** — memory bootstrap 与 skill index 是"每轮 LLM 都要看见但又不属于会话历史"的横切信息；plugin 是它们唯一合法的注入通道
- **奥卡姆剃刀** — v1 不做 embedding / 不做 `.index.json` / 不做 fsync 分级 / 不做依赖图
- **状态归调用方** — plugin 不进 thread.messages；MEMORY.md / SKILL.md 是磁盘上的 single source of truth
- **Rule of three** — 两个 plugin 共享"system 末尾注入 + mtime cache + 失败降级"模式；不抽公共基类（只有两个实例，不到提取阈值）

## 四、交付范围

### 4.1 Framework 改动

| 项 | 现状 | 目标 |
|---|---|---|
| `Plugin` 接口 | `{ name, hooks }` | `{ name, hooks, tools?: readonly Tool[] }` |
| `createAgent` 入口校验 | `validateCheckpointer(checkpointer)` | + `validatePlugins(plugins, configTools)` |
| Tool 合并 | 直接用 `config.tools` | `[...config.tools ?? [], ...plugins.flatMap(p => p.tools ?? [])]` |
| Agent 持有 tools | `this.#tools = config.tools ?? []` | `this.#tools = mergedTools`（构造期一次） |
| 重名检查 | 无 | fail-fast：`Tool name collision: '${name}' declared by both '${prev}' and plugin '${p.name}'` |
| `Object.freeze(plugin.tools)` | — | 不做（TS `readonly` 表达 + 文档约定即可）|

**`validatePlugins` 签名与位置**：

```ts
// packages/framework/src/plugin.ts

export function validatePlugins(
  plugins: readonly Plugin[],
  configTools: readonly Tool[] = [],
): readonly Tool[] {
  const seen = new Map<string, string>();  // name → source
  for (const t of configTools) seen.set(t.name, "config.tools");
  for (const p of plugins) {
    for (const t of p.tools ?? []) {
      const prev = seen.get(t.name);
      if (prev) {
        throw new Error(
          `Tool name collision: '${t.name}' declared by both '${prev}' and plugin '${p.name}'`,
        );
      }
      seen.set(t.name, `plugin:${p.name}`);
    }
  }
  return [
    ...configTools,
    ...plugins.flatMap((p) => p.tools ?? []),
  ];
}
```

在 `createAgent` 入口调用：

```ts
const allTools = validatePlugins(plugins, config.tools);
// allTools 被 Agent 类持有，传给底层 L2 run()
```

**对称性**：与 [`validateCheckpointer`](../architecture/04-checkpointer.md#能力分层capability-detection) 同款样式（启动期检查、抛错带可操作 message）。

**`definePlugin` 签名更新**：

```ts
export function definePlugin(definition: {
  name: string;
  hooks: PluginHooks;
  tools?: readonly Tool[];   // ← 新增，透传
}): Plugin {
  return definition;
}
```

### 4.2 Package: `@my-agent-team/plugin-fs-memory`

**包路径**：`packages/plugin-fs-memory/`

**依赖**：
- `@my-agent-team/core`（Tool / Message 类型）
- `@my-agent-team/framework`（Plugin / definePlugin / HookContext 类型）
- `node:fs/promises` + `node:path`（无外部 npm 依赖）

**导出**：
```ts
export interface FsMemoryOptions {
  dir: string;
  enableWrite?: boolean;   // 默认 true
  searchLimit?: number;    // 默认 5
}

export function fsMemoryPlugin(options: FsMemoryOptions): Plugin;
```

**目录布局（运行时在磁盘上）**：

```text
${dir}/
├── MEMORY.md
└── facts/
    ├── 2026-06-05T14-17-06-123-用户偏好简洁.md
    └── ...
```

**注意**：v1 **不维护 `.index.json`**（M5 grill Q9 决议）。每次 `memory_search` 全扫 `facts/*.md` + mtime cache。

**目录与文件初始化策略**（M5 grill Q7）：

| 情况 | 行为 |
|---|---|
| `dir` 不存在 | `mkdir -p ${dir}/facts`（自动创建） |
| `MEMORY.md` 缺失 | 视为空，跳过注入；**不**自动创建空文件 |
| `MEMORY.md` 为空 | 跳过注入 |
| `MEMORY.md` 读失败 | `logger.warn` + 跳过注入 |

**注入实现**（在 `beforeModel`）：

```ts
async beforeModel(ctx, messages) {
  let memContent: string;
  try {
    memContent = await readMemoryWithMtimeCache();
  } catch (err) {
    ctx.logger.warn("fs-memory: read failed, skipping injection", err);
    return [...messages];
  }
  if (!memContent) return [...messages];

  const systemIdx = messages.findIndex((m) => m.role === "system");
  if (systemIdx < 0) {
    ctx.logger.warn(
      "fs-memory: no system message found, skipping memory injection. " +
        "Use createAgent({ systemPrompt }) to enable.",
    );
    return [...messages];
  }

  const sys = messages[systemIdx];
  const newSys = {
    ...sys,
    content: `${sys.content}\n\n<memory>\n${memContent}\n</memory>`,
  };
  return [
    ...messages.slice(0, systemIdx),
    newSys,
    ...messages.slice(systemIdx + 1),
  ];
}
```

**Tools**：

| Tool | 入参 | 返回 | 副作用 |
|---|---|---|---|
| `memory_read` | `{ path?: string }` | MEMORY.md 全文 或 fact 全文 | 无 |
| `memory_write` | `{ content: string; tags?: string[] }` | `{ path: string }` | 在 `facts/` 写一个新 md 文件 |
| `memory_search` | `{ query: string; limit?: number }` | `Array<{ path, title, tags, snippet }>` | 无 |

**`memory_read` 语义**（M5 grill Q5/Q6）：

| 情况 | 返回 |
|---|---|
| `memory_read()` 无 path，MEMORY.md 不存在 | `{ content: "" }` |
| `memory_read()` 无 path，MEMORY.md 存在 | `{ content: <全文> }` |
| `memory_read({ path })` 文件存在 | `{ content: <全文> }` |
| `memory_read({ path })` 文件不存在 | `{ content: "Fact not found: <path>", isError: true }` |
| `memory_read({ path })` 越界 | `{ content: "Path escapes memory dir", isError: true }` |

path 是 `memory_search` / `memory_write` 返回值的字面回传，内部 `path.resolve(dir, path)` + `startsWith` 穿越校验，不接受绝对路径。

**`memory_search` 算法**（M5 grill Q8 = A）：

```ts
function search(query: string, limit: number) {
  const q = query.toLowerCase();
  const facts = await loadAllFactsWithMtimeCache();
  const scored = facts
    .map((f) => {
      const tagHit = (f.tags ?? []).some((t) => t.toLowerCase().includes(q)) ? 3 : 0;
      const titleHit = f.title.toLowerCase().includes(q) ? 2 : 0;
      const bodyHit = f.body.toLowerCase().includes(q) ? 1 : 0;
      return { fact: f, score: tagHit + titleHit + bodyHit };
    })
    .filter((s) => s.score > 0)
    .sort((a, b) => b.score - a.score)
    .slice(0, limit);
  return scored.map(({ fact }) => ({
    path: fact.path,
    title: fact.title,
    tags: fact.tags ?? [],
    snippet: fact.body.slice(0, 200),
  }));
}
```

- 纯 substring match（lowercase），不分词，不 stem
- 中文天然支持（substring 不需要分词器）
- 无匹配返回 `[]`（M5 grill Q7）
- 依赖零

**`memory_write` 文件命名**（M5 grill Q10 = B + 兜底 A）：

```ts
async function memoryWrite({ content, tags }) {
  const ts = new Date()
    .toISOString()
    .replace(/[:.]/g, "-");           // 2026-06-05T14-17-06-123Z
  const slug = slugify(content.slice(0, 40));
  let filename = `${ts}-${slug}.md`;
  let filepath = path.join(dir, "facts", filename);
  let n = 2;
  while (await exists(filepath)) {    // 毫秒 ts 仍撞 → 后缀 -2/-3/...
    filename = `${ts}-${slug}-${n}.md`;
    filepath = path.join(dir, "facts", filename);
    n++;
  }

  // title 派生（M5 grill Q4）：首行剥 # 前缀，≤80 字
  const firstLine = content.split('\n').find((l) => l.trim()) ?? '';
  const title = firstLine.replace(/^#+\s*/, '').slice(0, 80);

  const frontmatter =
    `---\n` +
    `ts: ${ts}\n` +
    `title: ${JSON.stringify(title)}\n` +
    `tags: ${JSON.stringify(tags ?? [])}\n` +
    `---\n`;
  await fs.writeFile(filepath, frontmatter + content);  // 不 fsync (Q11)
  factsCache = null;   // 主动失效 cache (Q11)
  return { path: filepath };
}
```

**不 fsync**（M5 grill Q11 = A）：与 [fileCheckpointer 同款纪律](../architecture/04-checkpointer.md#filecheckpointer)，memory 整体是增益不是契约。

**mtime cache 策略**：

```ts
let memCache: { content: string; mtime: number } | null = null;
let factsCache: { facts: Fact[]; mtime: number } | null = null;

async function readMemoryWithMtimeCache(): Promise<string> {
  const memPath = path.join(dir, "MEMORY.md");
  try {
    const stat = await fs.stat(memPath);
    if (!memCache || memCache.mtime !== stat.mtimeMs) {
      memCache = {
        content: await fs.readFile(memPath, "utf-8"),
        mtime: stat.mtimeMs,
      };
    }
    return memCache.content;
  } catch (err) {
    if (err.code === "ENOENT") return "";
    throw err;
  }
}

async function loadAllFactsWithMtimeCache(): Promise<Fact[]> {
  const factsDir = path.join(dir, "facts");
  const dirStat = await fs.stat(factsDir);
  if (factsCache && factsCache.mtime === dirStat.mtimeMs) return factsCache.facts;
  const files = await fs.readdir(factsDir);
  const facts = await Promise.all(
    files.filter((f) => f.endsWith(".md")).map(loadFact),
  );
  factsCache = { facts, mtime: dirStat.mtimeMs };
  return facts;
}
```

注：`facts/` 目录 mtime 变化 = 文件增删；单文件内容修改不影响目录 mtime——但 v1 fact 是 append-only（`memory_write` 只新增不修改），所以这个 cache 失效策略正确。`memory_write` 后主动 `factsCache = null` 保证同 turn 强一致（M5 grill Q11）。

### 4.3 Package: `@my-agent-team/plugin-progressive-skill`

**包路径**：`packages/plugin-progressive-skill/`

**依赖**：
- `@my-agent-team/core`
- `@my-agent-team/framework`
- `gray-matter@^4.1.0`（M5 grill Q12 = ^4.1.0，~50KB，主流 + 0 配置）
- `node:fs/promises` + `node:path`

**导出**：
```ts
export interface ProgressiveSkillOptions {
  dir: string;
  maxCharsPerLoad?: number;  // 默认 8000
}

export function progressiveSkillPlugin(options: ProgressiveSkillOptions): Plugin;
```

**目录布局**（[Claude Code Skills 形态](../architecture/10-plugin-progressive-skill.md#二目录结构)）：

```text
${dir}/
├── pdf-extract/
│   ├── SKILL.md          # frontmatter: name + description; body 为详细指令
│   └── extract.py
├── docx-generate/
│   └── SKILL.md
└── ...
```

**目录初始化**（M5 grill Q8）：`dir` 不存在 → `logger.warn` + 注入空 index（不 auto-mkdir）；空目录 → 静默跳过。

| 情况 | 行为 |
|---|---|
| `dir` 不存在 | `logger.warn` + 跳过注入（用户配错了路径） |
| `dir` 存在但无子目录 | 静默跳过注入（用户还没装 skill，合法初始态） |
| `dir` 存在，某 skill 解析失败 | 该 skill `logger.warn` + 跳过，其他继续（allSettled） |
| `dir` 无读权限 | `logger.warn` + 跳过注入（IO 错误） |

**注入实现**：

```ts
async beforeModel(ctx, messages) {
  const skills = await loadSkillIndexWithMtimeCache();
  if (skills.length === 0) return [...messages];

  const systemIdx = messages.findIndex((m) => m.role === "system");
  if (systemIdx < 0) {
    ctx.logger.warn("progressive-skill: no system message, skipping injection");
    return [...messages];
  }

  const indexBlock = renderIndex(skills);
  const sys = messages[systemIdx];
  const newSys = {
    ...sys,
    content: `${sys.content}\n\n${indexBlock}`,
  };
  return [
    ...messages.slice(0, systemIdx),
    newSys,
    ...messages.slice(systemIdx + 1),
  ];
}

function renderIndex(skills: SkillMeta[]): string {
  const lines = skills.map((s) => `- **${s.name}**: ${s.description}`);
  return `<available-skills>
${lines.join("\n")}

Call \`skill_load(name)\` to load the full instructions for a skill before using it.
</available-skills>`;
}
```

**Skill 索引扫描**（M5 grill Q14 = allSettled）：

```ts
async function loadSkillIndex(): Promise<SkillMeta[]> {
  const entries = await fs.readdir(dir, { withFileTypes: true });
  const dirs = entries.filter((e) => e.isDirectory());
  const results = await Promise.allSettled(
    dirs.map((d) => loadOneSkillFrontmatter(path.join(dir, d.name))),
  );
  const skills: SkillMeta[] = [];
  for (let i = 0; i < results.length; i++) {
    const r = results[i];
    if (r.status === "fulfilled" && r.value) {
      skills.push(r.value);
    } else if (r.status === "rejected") {
      ctx.logger.warn(`skill '${dirs[i].name}' load failed`, r.reason);
    }
  }
  return skills;
}
```

单个 SKILL.md 解析失败：跳过 + warn + 继续其他。全有全无是反模式。

**`SkillMeta` 结构（含 bodyOffset）**（M5 grill Q9）：

```ts
interface SkillMeta {
  name: string;
  description: string;
  dir: string;
  skillMdPath: string;
  bodyOffset: number;   // scan 阶段记下，skill_load 不再调 gray-matter
}
```

`loadOneSkillFrontmatter` 用 `gray-matter` 解析后，`bodyOffset = raw.length - parsed.content.length`。

**Tool `skill_load`**：

| 入参 | 返回 |
|---|---|
| `{ name: string; offset?: number }` | `{ content: string }` body 片段（可能含 `[Truncated. Call skill_load('...', offset=N) to continue.]`） |

**截断策略**（M5 grill Q13 = B 段落边界）：

```ts
async function loadSkillBody(skillName: string, offset: number) {
  const skill = await findSkill(skillName);
  if (!skill) return { content: `Skill not found: ${skillName}`, isError: true };

  const raw = await fs.readFile(path.join(skill.dir, "SKILL.md"), "utf-8");
  const body = raw.slice(skill.bodyOffset);                // ← 不再调 gray-matter (Q9)
  const resolved = body.replaceAll("${SKILL_DIR}", path.resolve(skill.dir));

  if (offset >= resolved.length) {
    return { content: `Skill ${skillName} fully loaded.` };
  }

  const hardEnd = offset + maxCharsPerLoad;
  let realEnd: number;
  if (hardEnd >= resolved.length) {
    realEnd = resolved.length;
  } else {
    // 向后扫描到下一个段落边界，最多多读 500 字
    const lookahead = Math.min(resolved.length, hardEnd + 500);
    const tail = resolved.slice(hardEnd, lookahead);
    const breakIdx = tail.indexOf("\n\n");
    realEnd = breakIdx >= 0 ? hardEnd + breakIdx : hardEnd;
  }

  const chunk = resolved.slice(offset, realEnd);
  const hasMore = realEnd < resolved.length;
  const suffix = hasMore
    ? `\n\n[Truncated. Call skill_load('${skillName}', offset=${realEnd}) to continue.]`
    : "";
  return { content: chunk + suffix };
}
```

**`${SKILL_DIR}` 占位符**（M5 grill Q15 = 只 SKILL_DIR）：

仅替换 `${SKILL_DIR}` → skill 目录绝对路径。**不**支持 `${SKILL_NAME}` / `${MEMORY_DIR}` / `${HOME}` 等其他占位符——奥卡姆 + 边界清晰。

**mtime cache**：与 fs-memory 同款，cache key = `dir` 自身的 mtime。`dir` mtime 变化 = 子目录增删（添加新 skill 或删除 skill）；单 SKILL.md 内容修改不变 `dir` mtime——所以这个 cache 对 frontmatter 的修改不敏感。

> **取舍**：单 SKILL.md 编辑后 description 不会立即在下一轮注入更新。但 description 是低频修改字段（一旦定型不动）；body 修改不影响注入（body 走 `skill_load` 实时读）。可接受。如果用户要强刷，删 + 加回即可触发 dir mtime 变化。

**两个 plugin 各自有 cache.ts，不抽公共**（M5 grill Q10）：

| | fs-memory cache 形状 | progressive-skill cache 形状 |
|---|---|---|
| key | 单文件 mtime (`MEMORY.md` stat.mtimeMs) | 目录 mtime (`dir` stat.mtimeMs) |
| value | 文件内容字符串 | `SkillMeta[]` 数组 |
| 失效触发 | 文件内容变更 | 子目录增删 |
| facts/ 二级 cache | facts/ 目录 mtime → `Fact[]` 数组 | 不适用 |

两个 cache 形状不同，抽公共基类反而要参数化 4 个东西，成本超过手抄。Rule of three——第三个 plugin 出现时再抽。

## 五、技术契约不做（invariant）

这里只列**永久性技术契约**，不是 milestone 排期。这些"不做"是设计的一部分，跨 milestone 都不会变。

- **不内置 embedding** — memory v1 keyword search；embedding 留独立适配包
- **不维护 `.index.json`** — `memory_search` 全扫 + mtime cache，避免双写
- **不 fsync** — 与 fileCheckpointer 同款纪律
- **plugin 不动态加 tool** — `Plugin.tools` 是静态字段，`createAgent` 构造期合并一次
- **重名 tool 不静默覆盖** — fail-fast，错误消息明确指出冲突 source
- **不支持深层 skill 目录** — 固定 `${dir}/*/SKILL.md` 一层嵌套
- **不缓存 skill body** — 只缓存 frontmatter index；body 每次 `skill_load` 重读
- **只 `${SKILL_DIR}` 一个占位符** — 不引入其他变量替换
- **不做 skill 依赖图** — 让 LLM 自己读 A 后判断要不要 load B
- **`MEMORY.md` 缺失视为空** — plugin 不自动创建空文件
- **`skill dir` 不 auto-mkdir** — skill 是用户/harness 管理的资产
- **gray-matter 是 progressive-skill 的唯一外部 npm 依赖** — fs-memory 零外部依赖
