# M18.8 Spec — ColumnConfig 编辑 UI（把"列可配置"接上前端入口）

> **Status:** Spec → Ready for implementation.
> **Baseline commit:** `11e64516`（M18.1~M18.7 全落）。
> **决策（2026-06-20，本轮 grilling）:**
> - **D1→纯前端棒，零后端**：ColumnConfig 的 CRUD 后端契约全部已就位且被 reactor 消费中。缺的只有「人怎么编辑这些行」的 UI。本棒**一行后端都不改**。
> - **D2→状态选择器只露"可自动推进"的列**：排除 `draft`（`draft→planned` 是人工拖拽触发，reactor 永不读它的 ColumnConfig）+ 排除 `in_review`（`HUMAN_GATES`，闸门列由人裁决，配了也不会自动走）。只剩 `planned` / `in_progress`。
> - **D3→入口挂在 Project 卡片，不挂 issues 看板**：ColumnConfig 是 **per-project** 的（`upsert` key = `(projectId, status)`），归属 Project。
> - **D4→agent 选择器过滤 archived**：picker 用 `api.listAgents()`，前端过滤 `archivedAt != null`。
> - **D5→模板编辑给变量提示，不做校验/补全**：`promptTemplate` 用 `<textarea>`，下方静态列出可用变量（`{{title}}` / `{{issueId}}` / `{{deliverables.<kind>.fields.<key>}}` / `{{deliverables.<kind>.ref}}`）。
> - **Q1→Sheet 组件**：编辑表单用 Sheet（slide-over from right），非 Dialog。
> - **Q2→B**：默认 placeholder `"Complete the task: {{title}}"`。
> - **Q3→B**：`fieldClass`/`labelClass` 抽到 `lib/form-styles.ts`。

## 验收标准
- **A1**：Project 卡片 hover 可见「列配置」按钮，点击弹出 Sheet
- **A2**：面板列出 `计划中` / `开发中` 两个可配列，不出现 `草稿` / `待 Review` / `已完成`
- **A3**：能新增/覆盖配置，保存后 toast 成功
- **A4**：同列再保存 → 覆盖，不新增行
- **A5**：agent 下拉不含已归档 agent
- **A6**：删除列配置 → 内联二次确认 → toast 成功
- **A7**：404/400/500 经 toast 反馈
- **A8**：`bun typecheck` + `bun lint` 通过；`git diff apps/backend` 为空
