## 交付

### Framework (`packages/framework`)

**`plugin.ts`** — 新增 `beforeRun`/`beforeStop` 钩子 + `StopDecision` 类型 + `HookContext.emit`:
- `beforeRun`: loop 外前置，每 run 一次，transformer
- `beforeStop`: 模型无 tool_use 时触发，可否决停止（`{continue:true,reason}`）
- `emit`: 可选事件出口，插件借此推送 `todo_update` 等 AgentEvent

**`create-agent.ts`** — 新增 `todo_update` AgentEvent + `maxForceContinues` run 选项 + 聚合器 + 续跑门:
- `fireBeforeRun`/`fireBeforeStop` 聚合器（仿现有 fire 模式，fail-open）
- `run()` 在 push user 后调用 `fireBeforeRun`（transformer 替换 messages）
- `runLoop` 在 `toolUses===0` 时先过 `fireBeforeStop`；continue 时注入 reason 并续跑
- `maxForceContinues` 默认 3，为 0 关终止门
- `executeOne` 在 tool 执行后 drain `pendingEvents`（`ctx.emit` 推送的事件）

**`checkpointer.ts`** — 新增审计事件枚举: `force_continue`, `plan_injected`, `verify_round`, `verify_exhausted`

### Plugin: `@my-agent-team/plugin-task-guard`（新包）

**`taskGuardPlugin(opts)`** — 一个插件收束三处关切:
- `beforeRun`: `generatePlan` 拆任务 → 冻结 todos（全 pending）→ emit 初始快照
- `beforeModel`: 只读注入 `<todo>` 到 system 末尾
- `beforeStop`: 热层两道确定性门 — `unresolvedToolErrors` + todo 完成度
- `todo_write` 工具: 模型翻牌（只改 status，不增删 step），closure 绑定 threadId

全程 fail-open：拿不到 model / plan 失败 / validator 抛错 → 降级放行。

### Harness: `verificationGuidance()`

`packages/harness/src/verify.ts` — 冷审轮注入的 guidance。`reflect.ts` 的姐妹文件。

### Runner: 冷审 fork 循环

`packages/runner-stdio/src/entry.ts`:
- 主 run 后 → 冷审 fork 循环（`maxVerifyRounds` 默认 2）
- 冷审跑 `verificationGuidance()` + `maxForceContinues:0`
- 裁决 `complete=false` → `agent.run(missing)` 回流主线
- 冷审/reflection 后 → inline reflection（`maxForceContinues:0`）
- 冷审失败 → fail-open 放行

### Web UI: Todo 展示

- `conversation-reducer.ts`: 新增 `todos` state + `todo/update` action（全量快照替换）
- `useConversation.ts`: `/events` durable 通道监听 `todo_update` 事件
- `TodoPanel.tsx`: 新组件，空数组不渲染，显示 done/total + 勾选态
- `ConversationCanvas.tsx`: Header 下渲染 `<TodoPanel>`

**零后端改动**: `todo_update` 是 durable AgentEvent，经 supervisor `else` 分支自动持久化到 EventLog，eventStream 的 subscribe（live）和 read（replay）两路自动覆盖。
