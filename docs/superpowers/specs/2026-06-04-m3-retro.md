## Actual vs Spec

### Matched

- createAgent async，checkpointer.load 在构造时
- Agent.run 自动插 systemPrompt（仅首次）
- Agent.fork 复用 config，默认空 messages
- beforeModel 管道链，beforeTool skip 语义
- after* 吞错，before* 传播
- checkpointer.save 在 afterTool + 纯文本 return 前
- signal.throwIfAborted 早抛

### Differences

1. ~~**并发保护未实现**~~ — 已修复。`#running` guard + fail-fast 抛错
2. ~~**`createAgentInternal` 重复**~~ — 已修复。`createAgent` 委托 `createAgentInternal`，消除重复
3. ~~**`fork` 默认空 messages**~~ — 已修复。`fork()` 现在 `structuredClone(thread.messages)`
4. ~~**`after*` 吞错无日志**~~ — 已修复。所有 `after*` 和 `checkpointer.save` 错误都有 `console.warn`
5. `fileCheckpointer` 单文件限制保留（Checkpointer 接口本身支持多后端）
6. `Agent.run` 签名不跨包复用内联类型（非阻塞，M4 可改）

## Lessons

- L3 不调 L2 `run()` 是正确选择——L2 的 generator 太封闭，无法在中间 fire hooks
- `collectStream` 复用合理，不需要在 L3 重写 delta 累积
- createAgentInternal 重复是信号——应早点 refactor，不等 Rule of Three
- Checkpointer 独立字段 + Plugin 数组——比 LangChain 的 monolithic config 清晰很多
- fileCheckpointer 的单文件限制是实现简化而非设计限制
- 并发保护只需 4 行代码，应第一天实现
