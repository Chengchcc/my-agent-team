# Lobster v2.1 — Architectural Governance Spec

**Date**: 2026-05-26
**Status**: Draft
**Scope**: Typed CapabilityMap (G1), ContractBus unification (G2), concept cleanup + glossary (G3).
**Mode**: Break-change. 3 sequential PRs.

## PR G1 — Typed CapabilityMap

Add `src/kernel/capability-map.ts` with `CapabilityMap` interface mapping all ~25 capability keys to their types. Update `ExtensionRegistry.get<K>(key)` to be typed. Update `ExtensionApplyResult.provide` to be typed. All 45 call sites drop explicit `<X>` generic.

## PR G2 — Unify event flow on ContractBus

Delete `BusPort`. Make `ContractBus` the single emit surface for extensions. DataPlane moves to producer-side `dp.register(eventType, mapper)`. `emit` returns `Promise<void>`.

## PR G3 — Concept cleanup & glossary

Add `domain/anchor.ts` with `Anchor` discriminated union and `anchorToSessionId()`. Ban sentinels (`'main'`, `'unknown'`, `''`). Promote routingTable to `application/routing/`. Add `docs/architecture/glossary.md`.
