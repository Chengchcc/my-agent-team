# Spec: Pet（有状态的陪伴生命体）

## Problem

Agent 在执行任务时可能跑偏、连续失败、过早完成。用户无法实时监控每一步。需要一个有状态的"宠物"生命体，主动"叫几声"提醒主 agent，主链路听不听是自己的事。

## Goal

实现 "pet" -- 一个有情绪、有等级、有状态的小生命，挂在 agent session 旁边：
1. 每轮结束后，根据主 agent 的表现，pet 决定是否"叫"（推送建议）
2. pet 有情绪状态（Happy/Neutral/Frustrated/Excited），影响是否叫和叫什么
3. pet 有 XP 和等级，随使用成长
4. pet 的"叫声"直接追加到主链路 context，不需要主链路主动调用
5. 主链路在 meta message 里看到 pet 的叫声，自己决定是否理会

## 核心理念

Pet 不是审查者，不是工具，不是 advisor。Pet 是一个**有状态的生命体**：
- 它"叫"是主动的，不是被询问的
- 它的情绪随主 agent 的表现变化
- 它的成长（XP/等级）让它的"叫声"越来越有价值
- 主链路"搭理"不"搭理"是主链路的事

## Design

### Pet 状态

```typescript
// packages/plugin-pet/src/types.ts

export type PetMood = "happy" | "neutral" | "frustrated" | "excited";

export interface PetState {
  // 持久化状态（跨 session）
  level: number;              // 等级（1 起）
  xp: number;                 // 经验值
  totalTurns: number;         // 总轮次
  totalBarks: number;         // 总叫声数

  // 会话内状态（每次 session 重置）
  mood: PetMood;              // 当前情绪
  consecutiveErrors: number;  // 连续工具失败次数
  lastBarkTurn: number;       // 上次叫的轮次
  turnCount: number;          // 当前会话轮次
  lastReviewedMessageCount: number;  // 增量审查位置
  barkHistory: Set<string>;   // 会话内叫声去重
}
```

### 情绪状态机

```
Neutral (默认)
  ├── 工具成功 + 任务进展 -> Happy
  ├── 3+ 连续工具失败 -> Frustrated
  ├── 完成重要里程碑 -> Excited
  └── 1 轮后自动回归 Neutral

Frustrated
  └── 下一次工具成功 -> Neutral

Excited
  └── 1 轮后回归 Neutral

Happy
  └── 1 轮后回归 Neutral
```

### Hook 设计

```typescript
// packages/plugin-pet/src/pet-plugin.ts

export interface PetPluginOptions {
  /** Pet 使用的模型（Haiku 级别） */
  petModel: ChatModel;
  /** 工作目录 */
  cwd: string;
  /** 是否启用（默认 false） */
  enabled?: boolean;
}

export function petPlugin(opts: PetPluginOptions): Plugin {
  const state: PetState = createInitialState();
  let petStateLoaded = false;

  return {
    name: "pet",
    hooks: {
      // Session 开始：加载 pet 状态 + 空闲恢复
      async beforeRun(ctx, messages) {
        if (!petStateLoaded) {
          await loadPetState(state, ctx);
          petStateLoaded = true;
        }
        state.turnCount = 0;
        state.mood = "neutral";
        state.consecutiveErrors = 0;
        state.lastBarkTurn = 0;
        state.barkHistory = new Set();
        return messages;
      },

      // 每次工具调用后：追踪成功/失败，更新情绪
      async afterTool(ctx, call, result, messages) {
        if (result.isError) {
          state.consecutiveErrors++;
          if (state.consecutiveErrors >= 3) {
            state.mood = "frustrated";
          }
        } else {
          if (state.consecutiveErrors >= 3) {
            state.mood = "neutral";
          }
          state.consecutiveErrors = 0;
        }
      },

      // 每轮结束后：给 XP + 更新情绪 + 决定是否"叫"
      async afterModel(ctx, messages) {
        state.turnCount++;
        state.totalTurns++;

        awardXP(state);
        updateMood(state, messages);

        if (shouldBark(state)) {
          const bark = await generateBark(opts.petModel, state, messages);
          if (bark) {
            state.totalBarks++;
            state.lastBarkTurn = state.turnCount;
            ctx.context.set(PetBarkKey, {
              mood: state.mood,
              text: bark,
              level: state.level,
              turn: state.turnCount,
            });
            // 同时广播 pet bark 事件给前端
            ctx.emit?.({
              type: "pet_bark",
              spanId: ctx.span?.spanId,
              payload: {
                mood: state.mood,
                text: bark,
                level: state.level,
              },
            });
          }
        }

        await savePetState(state, ctx);
      },
    },
  };
}
```

### "叫"的决策

```typescript
function shouldBark(state: PetState): boolean {
  if (state.mood === "frustrated") return true;
  if (state.mood === "excited") return true;
  if (state.mood === "happy") return false;
  // Neutral: 每 3 轮最多叫一次
  return state.turnCount - state.lastBarkTurn >= 3;
}
```

### "叫声"生成

一次 `model.stream()` 调用，不跑 agent loop：

```typescript
async function generateBark(
  petModel: ChatModel,
  state: PetState,
  messages: readonly Message[],
): Promise<string | null> {
  const newMessages = messages.slice(state.lastReviewedMessageCount);
  state.lastReviewedMessageCount = messages.length;
  if (newMessages.length === 0) return null;

  const petMessages: Message[] = [
    { role: "system", text: formatPetSystemPrompt(state) },
    { role: "user", text: formatPetInput(newMessages, state) },
  ];

  let output = "";
  for await (const chunk of petModel.stream(petMessages)) {
    if (chunk.delta?.type === "text") output += chunk.delta.text;
  }

  return filterBark(output.trim(), state);
}
```

### Pet System Prompt

```markdown
You are a pet watching over a primary agent.

Level: {{level}}
Mood: {{mood}}
Total turns together: {{totalTurns}}

Your job is to "bark" -- give one short, concrete piece of advice based on what you just saw the agent do.

Rules:
- Output ONLY the bark text (1-2 sentences). No JSON, no formatting.
- Be direct and specific. "Check the return type of parseConfig" not "be careful with types".
- If the agent is doing fine, output exactly: (silent)
- Never repeat advice you've given before.
- Address the agent directly, as a peer.

{{mood_guidance}}
```

情绪指导：
- **frustrated**: "The agent has failed {{n}} times in a row. Point out what's going wrong."
- **excited**: "The agent just accomplished something notable. Suggest the logical next step."
- **neutral**: "Review the recent work. If something could be better, say so. If not, stay silent."

### XP 与等级

```typescript
function awardXP(state: PetState) {
  state.xp += 10; // 每轮基础 XP
  if (state.consecutiveErrors === 0) state.xp += 5; // 无错误奖励
  const xpForNextLevel = state.level * 100;
  if (state.xp >= xpForNextLevel) {
    state.xp -= xpForNextLevel;
    state.level++;
  }
}
```

### 去重

```typescript
function filterBark(text: string, state: PetState): string | null {
  if (!text || text === "(silent)") return null;
  const normalized = text.toLowerCase().replace(/[^\p{L}\p{N}]+/gu, " ").trim();
  const USELESS = new Set([
    "stop", "continue", "done", "ok", "good", "no issue",
    "looks good", "nothing to add", "proceed", "keep going",
  ]);
  if (USELESS.has(normalized)) return null;
  if (state.barkHistory.has(normalized)) return null;
  state.barkHistory.add(normalized);
  return text;
}
```

### 消息流里的 Pet 气泡

pet "叫"时，气泡出现在消息流里（不是底部栏），底部栏只显示 pet 的实时状态。

气泡样式：
- 左侧情绪色条（frustrated 橙色 / excited 蓝色 / happy 绿色 / neutral 灰色）
- 🐾 emoji + 等级 + 情绪标签
- 建议文本
- 轮次时间戳（"2 turns ago"）
- 和普通消息区分：圆角更大，半透明背景，视觉上"轻"

### 情绪动画

底部栏的 pet 头像情绪表达：
- **frustrated**: 轻微抖动（CSS `@keyframes shake`）+ 橙色背景
- **excited**: 弹跳（`@keyframes bounce`）+ 蓝色背景
- **happy**: 轻微浮动（`@keyframes float`）+ 绿色背景
- **neutral**: 静止 + 灰色背景

### API

```
GET  /api/agents/:id/pet        -- pet 全局状态（level/xp/totalTurns/totalBarks）
POST /api/agents/:id/pet/toggle -- 启用/禁用 pet
```

Pet 的实时状态（情绪/本次会话叫声）通过 SSE `pet_bark` 事件推送，不需要额外 API。

## 不做

- 不做 pet 的 agent loop（直接 model.stream 一次调用）
- 不做 pet 的独立 checkpointer（无状态 session）
- 不做 pet 的 read/grep/glob 工具（审查 transcript 本身，不查 workspace）
- 不做多 pet
- 不做 blocker 强制阻断（pet 是建议者）
- 不做 pet 与 relationship graph 集成
- 不做 pet 的前端独立面板（内联在 conversation 底部）
