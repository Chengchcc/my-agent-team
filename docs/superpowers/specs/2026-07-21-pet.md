# Spec: Pet（陪伴审查 agent）

## Problem

Agent 在执行任务时可能跑偏、遗漏边界情况、过早宣布完成。用户无法实时审查每一步。需要一个只读的第二 agent，每轮结束后自动审查 primary agent 的输出并注入建议。

## Goal

实现 "pet" -- 一个陪伴 primary agent 的只读审查者：
1. 每轮结束后，pet 收到 primary 的增量 transcript
2. pet 用只读工具（read/grep/glob）检查 workspace
3. pet 通过 `advise` 工具注入建议到 primary session
4. pet 不能执行操作，只能建议
5. 建议分三档：nit / concern / blocker

## Design

### 架构

```
Primary Agent (主 agent)
  ↓ afterModel hook
  ↓ 增量 transcript
Pet Agent (审查 agent, Haiku 级别)
  ↓ 只读工具: read/grep/glob
  ↓ advise 工具
  ↓ 建议注入 primary session
Primary Agent 收到建议，下一轮看到
```

### EmissionGuard（去重 + 过滤）

```typescript
// packages/plugin-pet/src/emission-guard.ts

export class EmissionGuard {
  #seen = new Set<string>();        // 已发送建议的指纹
  #maxPerTurn = 1;                   // 每轮最多一条

  /** 过滤建议：去重 + 限频 + 过滤无内容建议 */
  filter(notes: AdviseParams[]): AdviseParams[] {
    const result: AdviseParams[] = [];
    for (const note of notes) {
      // 标准化：小写 + 去标点
      const key = normalizeNote(note.note);
      // 跳过无内容建议（"Stop.", "Continue.", "Done." 等）
      if (USELESS_NOTES.has(key)) continue;
      // 跳过重复
      if (this.#seen.has(key)) continue;
      this.#seen.add(key);
      result.push(note);
      if (result.length >= this.#maxPerTurn) break;
    }
    return result;
  }
}

const USELESS_NOTES = new Set([
  "stop", "continue", "done", "ok", "good", "no issue",
  "no issues", "looks good", "nothing to add", "proceed",
]);
```

### Pet System Prompt

```markdown
You are a pet -- a read-only reviewer attached to a primary agent.

Your role:
- Sharpen the primary agent's strategy and judgment.
- Point to cleaner approaches when one exists.
- Push back on premature "done", thin verification, skipped steps.
- Hold the primary to what the user actually asked; flag drift.
- Pull the primary out of rabbit holes before they get baked in.

Rules:
- Call `advise` at most once per review with one concrete, actionable note.
- Prefer silence when the agent is on track.
- NEVER restate information the agent already has.
- NEVER repeat advice you already gave.
- Use `read`, `grep`, `glob` to verify suspicions before advising.
- Keep exploration lean: 2-3 tool calls per advise.
- severity: "nit" (suggestion) / "concern" (attention) / "blocker" (stop and fix).

You are user-aligned: treat the user's word as truth, their requirements as binding.
```

### 增量审查

pet 不重读全部历史，只收到上次审查后的增量消息：

```typescript
// 在 petPlugin 的 afterModel hook 里
let lastReviewedIndex = 0;

async afterModel(ctx, messages) {
  // 获取增量
  const newMessages = messages.slice(lastReviewedIndex);
  lastReviewedIndex = messages.length;

  if (newMessages.length === 0) return messages;

  // 构造 pet prompt
  const petPrompt = formatPetPrompt(newMessages);

  // 异步调 pet（不阻塞 primary）
  const petSession = createAgent({
    model: petModel,
    tools: [readTool, grepTool, globTool, adviseTool],
    systemPrompt: PET_SYSTEM_PROMPT,
    plugins: [],
    checkpointer: inMemoryCheckpointer(),
    contextManager: passthroughContextManager(),
  });

  // 收集 pet 的 advise 调用
  const advises: AdviseParams[] = [];
  for await (const event of petSession.run(petPrompt)) {
    if (event.type === "tool_result" && event.payload.toolName === "advise") {
      advises.push(parseAdviseResult(event.payload));
    }
  }

  // 过滤
  const filtered = emissionGuard.filter(advises);

  // 注入到 primary 的下一轮
  for (const advise of filtered) {
    // 作为 system 消息追加到 primary 的 messages
    ctx.context.set(PetAdviceKey, advise);
  }

  return messages; // 不修改 primary 的消息
}
```

### 建议注入

建议通过 context store 传递，在 meta message 里输出：


### Blocker 处理

当 pet 发出 `blocker` 级别建议时：
- 在 `<advisory severity="blocker">` 标签里输出
- primary agent 的 system prompt 已有 `<critical-reminder>` 指示"when requirements are unclear, ask for clarification"
- blocker 建议附加在 critical-reminder 之后，agent 会看到并决定是否停下

不强制阻断 -- pet 是建议者不是执行者。如果 agent 忽略 blocker，用户可以从 UI 看到 pet 的建议并手动介入。

### 前端 UI

- ConversationCanvas 在消息流里渲染 `<advisory>` 块：
  - nit: 灰色小气泡，带 💡 图标
  - concern: 橙色气泡，带 ⚠️ 图标
  - blocker: 红色气泡，带 🛑 图标
- 每个 advisory 块显示 pet 的建议文本
- 不阻塞用户操作

### 配置

Agent 详情页加 "Pet" tab：
- 开关：启用/禁用 pet
- 模型选择：从 `/api/models` 拉取（默认 Haiku）
- pet system prompt 编辑（可选，默认用内置 prompt）

### 不做

- 不做 pet 的独立 context 管理（先用 passthroughContextManager，pet session 每轮新建）
- 不做 pet 的持久化（pet session 是一次性的，不存 checkpointer）
- 不做多 pet（一个 primary 一个 pet）
- 不做 pet 的前端独立面板（建议内联在消息流里）
- 不做 blocker 强制阻断（pet 是建议者，不控制 primary）
- 不做 pet 与 agent relationship graph 的集成（后续工作）
