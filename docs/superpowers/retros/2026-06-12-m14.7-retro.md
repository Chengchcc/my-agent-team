| 计划 | 实际 | 偏离原因 |
|------|------|----------|
| RemoteCheckpointer RPC over transport | SQLiteCheckpointer via runner-state volume | grill 后决策：checkpoint 是 runner 执行恢复状态，不该跨 transport。边界更干净 |

## 二、设计决策回顾

### 做对的

1. **Checkpointer 不跨 transport**。这是 grill 后最重要的纠正——RemoteCheckpointer RPC 方案把 checkpoint payload 放进协议，边界混乱。改为 runner-local sqlite + durable runner-state volume 后，backend=生命周期、runner=执行恢复、AFS=文件抽象，三层边界清晰。

2. **Transport 抽象先于 socket 实现**。`RunnerTransport` 接口只定义 `send/onMessage/onClose`，`MemoryTransport` 用于测试，`SocketTransport` 用于部署。业务代码不依赖 socket path 或 `net.Socket`。

3. **复用现有 sqliteCheckpointer**。`checkpointer-sqlite` 包已有完整 6 方法实现 + migration ledger。不需要新写。迁移进 framework 后删了一个包。

4. **Mount table 最长前缀匹配作为一等能力**。`WorkspaceFS.#resolve` 纯函数，不依赖 transport、不依赖 daemon、不依赖 backend——测厚了 15 个用例。

5. **fork() 自动委派到 start()**。supervisor 同时支持 spawn 和 transport 两条路径，`fork()` 检测到 transport 时自动 delegate。过渡期零破坏。

6. **bootstrap 收窄到 WorkspaceFS**。从 `string | WorkspaceFS` 双路径改成只接受 `WorkspaceFS`，调用方负责 string→FS 适配。bootstrap 内部不再有 `readOrEmpty(path.join(...))` 的物理路径逻辑。

### 做错的 / 待改进

1. **inline reflection 是 M14.6 埋的雷**。M14.6 把 reflection 内联到 runner entry 里，导致 D19 污染 + 锁不释放 + 前端折叠错位。M14.7 修复了这个问题（删 inline reflection、reflection 归 daemon 控制环），但根本原因是 M14.6 的 inline 设计没有考虑"辅助 run 必须独立 runId"这个不变量。

2. **daemon agent factory 多次回退**。先写了 RemoteCheckpointer，再改成 AfsCheckpointer，再改成 SQLiteCheckpointer，最后发现已有包直接复用。如果先做完整的依赖探索（grep 全仓），省一轮重写。

3. **bun workspace symlink 问题**。新增 `workspace-fs` 包后 `bun install` 不自动创建 node_modules symlink，导致 typecheck 失败。手动 `ln -sf` 解决，但根因是 bun 的 workspace 链接策略与 turbo 的 `^build` 依赖关系不兼容。

4. **bootstrap 签名反复修改**。从 `(workspace: string)` → `(string | WorkspaceFS)` → `(WorkspaceFS, logger, workspaceLabel?)` → `(WorkspaceFS, logger)`。第三次参数 `workspaceLabel?` 是气味——它把展示关注点泄漏进了文件 I/O 函数。删掉后签名干净了。

## 八、如果重来一次

1. **先探索全仓依赖再写代码**。`checkpointer-sqlite` 包的存在是写到一半才发现的——如果开局 `grep -r "sqliteCheckpointer\|Checkpointer" packages/`，省一轮 RemoteCheckpointer 重写。

2. **grill 放在 spec 写完之后、代码写之前**。M14.7 的 grill 是在 spec 初版写完后进行的，纠正了 RemoteCheckpointer→SQLiteCheckpointer 方向。如果先写代码再 grill，浪费更大。

3. **Transport 抽象应该独立于 M14.7 先行**。`runner-protocol` 包是纯新增、零破坏，可以作为独立 PR 在 M14.7 之前合入，减少 M14.7 的 diff 体积。

4. **daemon skeleton 应该带最少测试**。`RunnerDaemon` 没有单元测试——因为它的 agent factory 依赖 harness，形成循环依赖。应该在 harness 接线前先写 memory transport + mock agent 的 daemon 测试。
