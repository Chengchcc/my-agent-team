## 二、设计决策回顾

### 做对的（spec 对齐度高）

1. **reducer 泛化而非重写**。M13.2 的 upsertAuthoritative / 对齐 / 去重 / 互斥逻辑完整继承，仅把匹配条件从 `role==="user"` 替换为 `sender.memberId === viewerMemberId`。16 个新 fixture 全部通过，0 回归。

2. **`stream/delta` 保留 `blockIndex`**（§4.0.2）。现网 `text_delta` payload 自带此字段，M14 保留透传，不丢语义。

3. **两种 SSE 解析分离**（§4.0.1）。`/runs/:id/events` 包层 `{type, payload}` vs `/conversations/:id/events` 扁平 ledger entry，hook 里两种解析路径独立，不与旧 payload 耦合。

4. **server component 直连快照**（§4.0.4）。首屏 bootstrap 走服务端 `fetchConversation`，避免 client 端 loading flash。`useQuery` 保留作为 fallback + 后台刷新。

5. **viewer 不可移除**（§4.7 不变量）。roster sidebar 的 human member 隐藏移除按钮，保证 `viewerMemberId` 永不出错。

### 待改进的（偏离 spec 的点）

1. **`addressedTo` 解析过于简单**。当前 `resolveAddressedTo` 返回唯一 agent memberId，多 agent 时返回 `[]`。后端 `postMessage` 用空的 `addressedTo` 走 `resolveTriggerTargets`（按 triggerMode 解析），当前 `mention` 模式会匹配所有 agent。多 agent 下第一次消息会触发所有 agent 的 run——这符合"广播"语义但未必是用户期望。后续需 UI 级 `@` 选择（M14 显式不做，留给未来）。

2. **`ThreadRow.kind` 未枚举 `"conversation"`**。NavRail/ThreadList 创建的 thread 的 `kind` 是 `"agent_thread"`，conversation 的 `kind` 是 `"conversation"`。但 ThreadList 仍用 `api.listThreads(agentId)` 取列表——conversation 不会出现在 agent thread 列表里。会话列表入口尚未做（M14 §十 L5 缺口表中 `GET /conversations` 列表端点缺失），这是个已知缺口。

3. **`Message` 类型保留但无用**。`api.ts` 的 `Message { role: "user"|"assistant"|"system" }` 类型已无 consumer，但保留以兼容旧代码路径。可在下次清理。

## 五、经验教训

1. **"保存相对量、存储绝对量"的设计威力**。把 `isSelf` 从存储层踢到渲染层后，所有身份问题（跨 surface、多 agent、IM）都变成了 `senderMemberId === viewerMemberId` 的比较——ledger 不再需要任何 per-surface 分支。这是从 thread-binary 到 conversation 的核心跃迁。

2. **纯函数 reducer 的迁移成本极低**。因为 M13.2 已经把渲染状态收敛到单一 reducer + 至多一条 draft，M14 的泛化仅在 `upsertAuthoritative` 和 `Action` 类型上做替换——结构完全不变。印证了 "alignment before migration" 是对的。

3. **Biome 全局格式化的副作用**。8 号 commit（`c81c71f`）是 biome format --write 的结果，改了 106 个文件，其中多数是我未编辑的文件。原因是本地 biome 版本与 CI 的配置可能有差。建议：更大范围的格式修改应单独 PR/MR 做，不混入 feature commit。

4. **hook 复杂度偏高**。`useConversation` 有 4 个 `useEffect`（snapshot bootstrap、ledger SSE、run stream、run events fallback）——逻辑清晰但 effect 数量多。后续可考虑抽 `useLedgerSSE` / `useRunStream` 独立 hook。
