## 经验教训

1. **导航单一事实源**：OpsTabs + NavRail 双重入口造成导航分裂。统一到 NavRail 后删掉 OpsTabs，运维永远不会困惑"我在哪"。
2. **agentName 是一人改、到处受益的 change**。后端 join 一次，前端 6 个组件免费拿到名字，比每个组件各自 `listAgents` 建 map 干净太多。
3. **Bug 7 方案 B→A 的条件**：当初移除 filter 参数是因为后端不支持。M16.2 把后端补齐后，前端立即恢复——这是一个典型的"先止血、后根治"的节奏。
4. **CSS token 收敛是纯机械活**：两套其实指向同一批颜色。补了 2 个 missing token 后，替换几乎只是 find-replace。chat 组件群留作独立任务是对的——避免动到刚稳定的会话渲染链路。
