## Spec revisions during implementation

1. **`stepStallTimeoutMs` removed from AgentSpec** (grill Q13): stays in BackendConfig only, used as reaper secondary check. Runner doesn't need it — if stuck, it naturally stops producing heartbeats.

2. **Heartbeat fallback timer removed** (grill Q11): progress heartbeat is single-source (per `sink.append()`). No unconditional fallback timer that would cause liveness false positives.

3. **Default values raised** (grill Q10): `heartbeatTimeoutMs` 20s→60s, new `stepStallTimeoutMs=300s`, `reaperIntervalMs=30s`.

4. **`bootstrap` export added to harness index** (implementation gap): was only used internally, now exported for e2e testing.

## Review findings

1. **Reflect=>existing test interaction**: Adding a second `agent.run()` to entry.ts broke all tests using naive mock agents. Fixed by guarding mock `run()` to yield only on first call. This is the right semantics — `agent.run()` is not idempotent in real usage either.

2. **Reaper tests DB sharing**: Multiple supervisor test instances share the same `events.db` file. Stale runs from prior tests accumulate. Fixed by using unique runIds per test + per-test assertions filtering by specific runId.

3. **`inMemoryEventLog.append` is async**: Reaper's setInterval callback was fire-and-forget without `await`, causing events to never land. Fixed by making `#reapStaleRuns()` async and awaiting the append.

4. **SQLite foreign_keys default OFF**: The `hardDelete` implementation requires `PRAGMA foreign_keys=ON` for CASCADE deletes to work. Added explicit PRAGMA in the transaction.

## Resolved (commit 7012167)

1. **Events.db cleanup in hardDelete**: Wired via 4 closure injections from main.ts composition root — `purgeWorkspace`, `purgeEventsForThreads`, `listThreadIds`, `assertNoActiveRun`. Same pattern as `threadSvc.cleanupCheckpoint`. Service never sees supervisor.

2. **Reflection turn pollution of main checkpoint**: Switched from `agent.run()` to `agent.fork()` for reflection. Fork creates temporary threadId — reflection messages never touch main thread's `checkpoint_messages`. Full context still visible (fork clones thread.messages).

## Known limitations

1. **Reaper "alive-but-stalled" branch lacks real-PID test coverage**: All reaper tests use fake PIDs (`99999`, `12345`) — `process.kill(pid,0)` always throws → "process dead" branch always taken. The stall-window branch (process alive + age within timeout+stall) is covered by P1 test added in follow-up.

2. **CLI `--rm` requires network**: The `--rm` command only works with `--backend` mode (remote HTTP). Local workspace-only mode has no agent management API.
